1.2 HTML: La struttura di una pagina web

HTML e il linguaggio che descrive la struttura del contenuto. Non e un linguaggio di programmazione — e un linguaggio di markup (marcatura).

<!DOCTYPE html>                    <!-- Dice al browser: "Questo e HTML5" -->
<html lang="it">                   <!-- Radice del documento -->
  <head>                           <!-- METADATI (non visibili) -->
    <meta charset="UTF-8">         <!-- Codifica caratteri -->
    <meta name="viewport"
          content="width=device-width">  <!-- Mobile responsive -->
    <title>VIO AI Orchestra</title>     <!-- Titolo tab browser -->
    <link rel="stylesheet"
          href="styles.css">       <!-- Foglio di stile -->
    <script src="app.js"
            defer></script>       <!-- JavaScript (defer = carica dopo) -->
  </head>

  <body>                           <!-- CONTENUTO VISIBILE -->
    <header>...</header>          <!-- Intestazione -->
    <nav>...</nav>                <!-- Navigazione -->
    <main>                         <!-- Contenuto principale -->
      <section>...</section>
      <article>...</article>
    </main>
    <footer>...</footer>          <!-- Pie di pagina -->
  </body>
</html>

XML (eXtensible Markup Language) e come HTML ma con regole piu rigide e tag personalizzati. Lo usi nei file .docx (che sono ZIP di XML), nei feed RSS, nelle configurazioni, e nelle API SOAP.

  <descrizione>App di orchestrazione AI locale</descrizione>

Differenza chiave: HTML ha tag predefiniti (<div>, <p>). XML ti permette di inventare i tuoi tag. Ma ogni tag aperto DEVE essere chiuso.

Cap. 2 — Cosa Contiene un Large Language Model (LLM)

Un LLM e un file (o insieme di file) che contiene:

(stessa struttura) | +----------------------------+ | ... x 80-128 blocchi (dipende dal modello) | [OUTPUT LAYER] (vettore --> probabilita su 100k+ token) | [SAMPLING] (sceglie il prossimo token) | OUTPUT: token successivo --> "Per" --> "creare" --> "un'" --> ...

2.3 Multi-Head Attention — come funziona davvero

Questa e la parte piu importante. L'attention e il meccanismo che permette al modello di capire le relazioni tra parole.

Formula matematica reale:

Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V

Dove:
  Q = Query (cosa sto cercando?)     Matrice [seq_len x d_model]
  K = Key (cosa posso offrire?)      Matrice [seq_len x d_model]
  V = Value (il contenuto effettivo) Matrice [seq_len x d_model]
  d_k = dimensione della chiave (es. 128)
  sqrt(d_k) = fattore di scala per stabilita numerica

Multi-Head: ripeti questo calcolo N volte in parallelo (es. 64 heads)
poi concatena i risultati e proietta con una matrice finale.
In parole semplici: Quando il modello legge "Il gatto si e seduto sul tappeto perche era stanco", l'attention gli permette di capire che "era stanco" si riferisce al "gatto" e non al "tappeto". Lo fa calcolando un punteggio di "rilevanza" tra ogni coppia di parole nella frase.

2.5 Cosa NON contiene un LLM

Miti da sfatare:

- Non contiene un "database" di testi. I testi di training vengono compressi nei pesi e poi distrutti. Il modello non puo "cercare" un documento specifico.

- Non ha "memoria" tra conversazioni (a meno che non si usi un database esterno come RAG).

- Non ha "coscienza" o "intenzioni". E un sistema di predizione statistica estremamente sofisticato.

- Non "ragiona" come un umano. Produce testo token per token in modo sequenziale, senza "pensiero interno" separato dall'output (con eccezione delle chain-of-thought come il tag <antml:thinking> di Claude).

Cap. 3 — I Token: Cosa Sono, Come Si Creano, Come Funzionano

3.1 Cos'e un token

Un token e un frammento di testo — non necessariamente una parola intera. Il tokenizer spezza il testo in pezzi gestibili.

Testo: "VIO AI Orchestra gestisce 12 modelli"

Tokenizzazione (BPE — Byte Pair Encoding):
["V", "IO", " AI", " Orchestra", " gest", "isce", " 12", " modelli"]

Token IDs: [86, 2394, 9147, 58234, 7823, 4521, 1105, 43892]

3.2 Come si costruisce un tokenizer (Byte Pair Encoding)

Il processo reale, step by step:

Passo 1: Parti dai singoli byte (256 simboli base)

Vocabolario iniziale: a, b, c, ..., z, A, ..., Z, 0-9, spazi, punteggiatura = ~256 token

Passo 2: Conta le coppie piu frequenti nel corpus

Corpus: "the cat the dog the cat"
Coppie piu frequenti: ("t","h")=3, ("h","e")=3, ("c","a")=2, ("a","t")=2

Passo 3: Fondi la coppia piu frequente in un nuovo token

"t" + "h" --> "th" (nuovo token 257)
Corpus diventa: "th e c a t th e d o g th e c a t"

Passo 4: Ripeti fino a raggiungere la dimensione desiderata

"th" + "e" --> "the" (token 258)
"c" + "a" --> "ca" (token 259)
"ca" + "t" --> "cat" (token 260)
... ripeti ~100.000 volte

Cap. 4 — System Prompt, User Input, Output: Il Ciclo Completo

4.1 Il flusso reale di una richiesta

[UTENTE digita messaggio] | [FRONTEND (React)] | POST /api/chat { message: "...", model: "claude" } | [BACKEND (FastAPI)] | Costruisce il PROMPT COMPLETO: +-------------------------------------------+ | SYSTEM PROMPT (istruzioni permanenti) | | "Sei VIO AI Orchestra..." | +-------------------------------------------+ | CRONOLOGIA CONVERSAZIONE | | [user]: messaggio 1 | | [assistant]: risposta 1 | | [user]: messaggio 2 | +-------------------------------------------+ | NUOVO MESSAGGIO UTENTE | | [user]: "Come funziona il RAG?" | +-------------------------------------------+ | [API PROVIDER] (Anthropic/OpenAI/Ollama) | Tokenizzazione --> Transformer --> Sampling | [STREAM DI TOKEN] "Il" "RAG" "(" "Retri" "eval" ... | [BACKEND riceve stream] | [SSE (Server-Sent Events) al FRONTEND] | [REACT aggiorna la UI in tempo reale]

4.2 System Prompt: cosa contiene e perche e cruciale

Il system prompt e l'insieme di istruzioni che definisce il comportamento dell'AI. E la prima cosa che il modello "legge" ad ogni richiesta.

// Esempio reale da VIO AI Orchestra (semplificato)
{
  "role": "system",
  "content": `Sei VIO 83 AI Orchestra, un assistente AI avanzato.

  COMPORTAMENTO:
  - Rispondi in italiano se l'utente scrive in italiano
  - Usa fonti verificabili quando possibile
  - Se non sai qualcosa, dillo chiaramente

  CAPACITA:
  - Cross-check: confronta risposte da piu modelli
  - RAG: cerca nel knowledge base locale prima di rispondere
  - Code generation: genera codice con spiegazioni

  LIMITI:
  - Non inventare citazioni o dati
  - Non dare consigli medici/legali senza disclaimer`
}

4.3 Come il modello processa il prompt

Fatto critico: Il modello NON "ricorda" le conversazioni precedenti. Ogni volta che mandi un messaggio, il backend invia l'INTERA cronologia della conversazione al modello. Per questo le conversazioni lunghe costano di piu e diventano piu lente — il modello deve rielaborare tutto da zero ogni volta.

4.4 Temperature e Sampling: come sceglie le parole

Dopo aver processato il prompt, il modello produce una distribuzione di probabilita su tutti i token possibili (~100K). La "temperature" controlla quanto e "creativo":

TemperaturaComportamentoUso ideale
0.0Sceglie SEMPRE il token piu probabile (deterministico)Codice, matematica, dati fattuali
0.3-0.5Leggermente variabile ma controllatoRisposte professionali, analisi
0.7-0.9Piu creativo, meno prevedibileScrittura creativa, brainstorming
1.0+Molto casuale, rischio di incoerenzaQuasi mai consigliato

Cap. 5 — Semantica Contestualizzata e il "Byplane" dell'AI

5.1 Embedding: come l'AI "capisce" il significato

Ogni token viene convertito in un vettore (lista di numeri). Token con significato simile hanno vettori vicini nello spazio multidimensionale.

"re"     --> [0.23, -0.45, 0.82, ..., 0.11]  (8192 numeri)
"regina" --> [0.25, -0.43, 0.79, ..., 0.13]  (VICINO a "re")
"tavolo" --> [-0.67, 0.12, -0.34, ..., 0.88] (LONTANO da "re")

La famosa analogia:
  vettore("re") - vettore("uomo") + vettore("donna") ≈ vettore("regina")

5.2 Contestualizzazione: perche "banco" cambia significato

Nei modelli moderni (transformer), il vettore di ogni token cambia a seconda del contesto:

"Sono andato in banco"   --> banco = istituto finanziario
"Il banco di scuola"     --> banco = mobile
"Banco di nebbia"        --> banco = massa compatta

L'attention mechanism ricalcola il vettore di "banco" guardando
tutte le altre parole della frase. Questo e cio che rende i
transformer superiori ai modelli precedenti.

5.3 Perche le virgolette cambiano l'output

Domanda di Vio: "Perche se inserisco un acronimo con le virgolette da un output diverso da senza virgolette?"

Risposta tecnica: Le virgolette cambiano il contesto semantico del token. Quando scrivi "RAG" con virgolette, il modello interpreta che stai citando un termine specifico, un nome proprio, o un acronimo tecnico. Senza virgolette, RAG potrebbe essere interpretato come parola comune o parte di una frase generica.

A livello di tokenizzazione: "RAG" produce token diversi da RAG — le virgolette sono token separati che influenzano l'attention su tutta la frase. Il modello ha visto durante il training che testo tra virgolette tende ad essere trattato come entita specifica.

Cap. 7 — Protocollo di Aderenza Totale 100x

Questo e il framework che hai definito. Lo analizzo e lo certifico come valido:

Valutazione professionale: Il tuo "Protocollo di Aderenza Totale 100x" e un framework di prompt engineering di livello avanzato. La struttura (Ruolo + Obiettivo + Contesto + Formato + Criteri + Politica di verita) e allineata con le best practice di prompt engineering documentate da Anthropic, OpenAI e Google DeepMind. E pronto per l'uso professionale.

7.1 Analisi dei punti di forza

Obiettivo unico non negoziabile — Riduce la "dispersione" dell'AI. Un modello con un solo obiettivo chiaro produce output 3-5x piu precisi rispetto a prompt vaghi.

Politica di verita — Il blocco "non inventare dati" con richiesta di fermarsi e chiedere e cruciale. Previene le allucinazioni.

Checklist di accettazione — Trasforma la valutazione dell'output da soggettiva a oggettiva.

A/B con raccomandazione — Gestisce i conflitti senza bloccare il lavoro.

7.2 Come implementarlo in VIO AI Orchestra

Questo framework va integrato direttamente nel system prompt dell'app. Il file src/services/ai/systemPrompt.ts e dove inserirlo:

Cap. 8 — VS Code: Come Recuperare la Chat Claude

8.1 Dove VS Code salva le chat

Le conversazioni Claude in VS Code sono salvate nel database interno dell'estensione:

Percorso macOS:
~/Library/Application Support/Code/User/globalStorage/
  anthropic.claude-code/
    chat-sessions/

8.2 Comandi per recuperare/riattivare

Esegui dal Terminale Mac:

# 1. Verifica che l'estensione Claude sia installata
code --list-extensions | grep -i claude

# 2. Se non c'e, reinstallala
code --install-extension anthropic.claude-code

# 3. Apri VS Code nel progetto
code ~/Projects/vio83-ai-orchestra

# 4. Dentro VS Code premi:
#    Cmd+Shift+P --> "Claude: Open Chat"
#    oppure
#    Cmd+Shift+P --> "Claude: Show History"

8.3 Se la chat non si riapre

Il problema piu comune dopo un aggiornamento di VS Code e che l'estensione perde il contesto. Soluzioni:

# Reset completo estensione (preserva la cronologia)
Cmd+Shift+P --> "Developer: Reload Window"

# Se ancora non funziona, reinstalla
code --uninstall-extension anthropic.claude-code
code --install-extension anthropic.claude-code

# Poi riapri:
code ~/Projects/vio83-ai-orchestra
NOTA: La chat Claude in VS Code NON mantiene il contesto tra sessioni a meno che tu non abbia salvato la conversazione esplicitamente. Ogni volta che chiudi VS Code, la chat riparte da zero. Per mantenere continuita, usa il file CLAUDE.md nel progetto — Claude lo legge automaticamente ad ogni nuova sessione.
